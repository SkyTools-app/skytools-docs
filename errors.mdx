## Error format

All errors return a consistent structure:

```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable description of what went wrong"
  }
}
```

## Error reference

| Code | HTTP Status | Description |
|---|---|---|
| `BAD_REQUEST` | 400 | Missing or invalid query/path parameters |
| `VALIDATION` | 400 | Request body failed validation (wrong types, missing required fields) |
| `UNAUTHORIZED` | 401 | Missing, invalid, or revoked API key |
| `FORBIDDEN` | 403 | Valid key but insufficient permissions |
| `LIMIT_REACHED` | 403 | Tier feature limit reached (e.g. max portfolio entries) |
| `NOT_FOUND` | 404 | Resource not found or route doesn't exist |
| `RATE_LIMITED` | 429 | Rate limit exceeded — check `Retry-After` header |
| `INTERNAL_ERROR` | 500 | Unexpected server error |
| `DB_ERROR` | 500 | Database operation failed |
| `UPSTREAM_ERROR` | 502 | Upstream API (Hypixel or Mojang) returned an error |
| `TIMEOUT` | 503 | Upstream service timed out |

## Common errors

### Missing API key

```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "API key is required. Get one at https://developers.skytools.app/dashboard"
  }
}
```

**Fix:** Include your API key via `X-API-Key` or `Authorization: Bearer` header. See [Authentication](/authentication).

### Invalid API key

```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or revoked API key."
  }
}
```

**Fix:** Verify your key starts with `sk_` and hasn't been revoked. Generate a new key in the [Dashboard](https://developer.skytools.app/dashboard).

### Missing required parameter

```json
{
  "success": false,
  "error": {
    "code": "BAD_REQUEST",
    "message": "item_id parameter is required"
  }
}
```

**Fix:** Check the endpoint documentation for required parameters.

### Rate limit exceeded

```json
{
  "success": false,
  "error": {
    "code": "RATE_LIMITED",
    "message": "Rate limit exceeded. Max 60 requests per minute."
  }
}
```

**Fix:** Wait for the duration in the `Retry-After` header. See [Rate Limits](/rate-limits) for optimization tips.

### Route not found

```json
{
  "success": false,
  "error": {
    "code": "NOT_FOUND",
    "message": "Route GET /v1/unknown not found. See GET / for available endpoints."
  }
}
```

**Fix:** Verify the endpoint URL against the [API reference](/endpoints/bazaar).

### Upstream timeout (stale cache)

<Warning>
When an upstream service times out, the API may return stale cached data (up to 30 minutes old) instead of an error. Always check `meta.stale` to detect this.
</Warning>

If stale cached data is available, the response succeeds with extra metadata:

```json
{
  "success": true,
  "data": { ... },
  "meta": {
    "cached": true,
    "api_version": "v1",
    "stale": true,
    "age_seconds": 245
  }
}
```

If no cached data is available, you'll receive a `503`:

```json
{
  "success": false,
  "error": {
    "code": "TIMEOUT",
    "message": "Upstream service timed out. Please try again."
  }
}
```

### Tier limit reached

```json
{
  "success": false,
  "error": {
    "code": "LIMIT_REACHED",
    "message": "Portfolio limit reached for your tier (15). Upgrade for more."
  }
}
```

**Fix:** Upgrade your subscription or remove existing entries.
